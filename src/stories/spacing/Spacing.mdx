    # Spacing
    Spacing is the invisible structure that gives your interface breathing room and order. A consistent spacing scale ensures predictable margins, paddings, and gaps across the system, helping designers and developers create balanced layouts, maintain hierarchy, and deliver intuitive user experiences.

    <br />
    <br />

    ## **The Spacing Scale**

    Our spacing scale follows a modular system, making it adaptable for anything from fine-grained component adjustments to large-scale layout design. Each value has a dedicated token, ensuring both internal component spacing and spacing between components follow the same design language.

    <SpacingList />

    <br />

    ## **Best Practices for Applying Spacing**
    **When working with spacing, keep these principles in mind:**
      - Use consistent padding and gap values across similar patterns.
      - Visually group related elements by keeping them close together.
      - Space conveys meaning — proximity implies relationship.
      - Apply generous spacing around primary actions or focal content to draw attention.

    ## **Implementation**
    Spacing is a fundamental part of maintaining visual rhythm and hierarchy in our design system.  
    We use a consistent spacing scale across all components and layouts to ensure predictable, harmonious spacing between elements.  

    The examples below demonstrate two key approaches to applying spacing in the system:  
      - **Flex-Based Spacing** – Ideal for evenly distributing space between items without extra wrappers or manual margins.  
      - **Layout Spacing** – Defines the structural spacing between major layout sections or page elements.

    ### **Flex-Based Spacing**
    Flex-based spacing automatically distributes consistent gaps between items in a row or column.  
    Our `Flex` component uses the spacing scale to apply uniform separation, working both horizontally and vertically without the need for custom CSS.
