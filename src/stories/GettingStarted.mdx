## **Configuration**

To fully utilize the design system, you need to configure your application to load the necessary styles, fonts, and providers.

### **1. Global Styles**

Import the core CSS files at the root of your application (e.g., `pages/_app.tsx`, `app/layout.tsx`, or `src/main.tsx`).

<Source language="css" dark code={`
import '@clera-ui/styles/globals.css'
`} />

### **2. Typography**

{config.title} relies on the **Satoshi** font family for its distinct look. Import the font CSS to ensure text renders correctly.

<Source language="css" dark code={`
/* Import the Satoshi font family (optional) */
import '@clera-ui/styles/satoshi.css'
`} />

### **3. Theme Provider**

Wrap your application with the `ThemeProvider` to enable dynamic theme switching and context access.

<Source language="tsx" dark code={`
import { ThemeProvider } from '@clera-ui'

export default function RootLayout({ children }) {
    return (
      <ThemeProvider defaultTheme="system" enableSystem>
        {children}
      </ThemeProvider>
    )
}
`} />

## **Architecture**

The design system is structured into three main layers:

- **Primitives**: Low-level building blocks like `Box`, `Text`, and `Flex` that handle basic layout and typography.
- **Components**: Atomic UI elements like `Button`, `Input`, and `Badge` that compose primitives.
- **Patterns**: Complex compositions like `Card`, `Form`, and `Dialog` that solve specific user problems.
